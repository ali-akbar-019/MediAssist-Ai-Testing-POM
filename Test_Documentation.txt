PROJECT OVERVIEW
--------------------------------------------------------------------------------
Project Name     : MediAssistAi
Version          : 0.0.1-SNAPSHOT
Java Version     : 11
Test Framework   : TestNG 7.12.0
Automation Tool  : Selenium Java 4.44.0
Reporting        : ExtentReports 5.1.2
Build Tool       : Maven (with Surefire plugin)

Description:
MediAssistAi is a pure test automation project (no production source code) that
tests a medical assistance web application covering symptom analysis, AI chat,
medicine lookup, hospital finder, OCR document scanning, reports, dashboards,
and user management.

================================================================================
                    TEST SUITE - DETAILED BREAKDOWN
================================================================================

All test case files reside in:
    src/test/java/testCases/

BaseClass.java (155 lines) provides shared infrastructure:
  - setup/browser launch (Chrome/Edge/Firefox via TestNG parameter)
  - tearDown / driver cleanup
  - login() / logout() authentication methods
  - waitForElementVisible(), waitForUrlContains()
  - captureScreen() for failure screenshots
  - logInfo() for ExtentReport + console logging

Additional utilities in src/test/java/utilities/:
  - XLUtility.java       - Excel read/write (Apache POI)
  - ExtentReportManager.java - HTML report management

================================================================================
                    KEY FRAMEWORK FEATURES
================================================================================

  1. Page Object Model (POM)
     - Each web page has a corresponding class encapsulating its elements
       and actions
     - All pages extend/use WebDriver from BaseClass

  2. BaseClass Utility Layer
     - Centralized browser setup (Chrome/Edge/Firefox)
     - Reusable login/logout with automatic retry on redirect
     - Selenium wait helpers (visibility, URL contains)
     - Screenshot capture on test failure
     - Logging to ExtentReport and console

  3. TestNG Annotations
     - @BeforeMethod / @AfterMethod for setup/teardown
     - @Test with groups (smoke, regression, edge-case)
     - dependsOnMethods for test sequencing
     - TestNG XML for suite configuration

  4. Data-Driven via config.properties
     - Centralized configuration (URL, credentials)
     - Easy environment switching

  5. Listeners
     - ExtentReport for rich HTML reporting
     - Automatic failure screenshots

  6. Cross-Browser Support
     - Chrome, Edge, and Firefox via TestNG parameter

                    NOTES

  - Login session is verified and automatically retried if the session

  - Edge-case tests use TestNG group "edge-case" for targeted execution
